\section*{Summary}

DbSaicle enables remediation-first security by making high and critical
findings actionable inside the developer workflow. Two tools power this
approach:

\begin{itemize}[leftmargin=*]
  \item \texttt{oss\_vulnerability\_scan} for dependency vulnerabilities
  \item \texttt{veracode\_pipeline\_scan} for static analysis findings
\end{itemize}

The scans are inputs. The value comes from a consistent remediation loop:
scan, triage, fix, rebuild, and rescan until findings are cleared or accepted.

\section*{Why This Matters}

\begin{itemize}[leftmargin=*]
  \item Fixes issues while context is fresh, reducing remediation time
  \item Cuts rework by addressing findings before release
  \item Increases release confidence through repeatable validation
\end{itemize}
